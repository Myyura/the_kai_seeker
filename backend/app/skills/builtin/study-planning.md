# 受験対策 学習計画ガイド

When helping users create study plans for graduate entrance exams:

## Planning Framework

### 1. Understand the Target (目標確認)
- Which school(s) and program(s)?
- Which exam session (summer/winter)?
- How many months until the exam?
- What is the user's current level?

### 2. Analyze Exam Requirements
- Use search_schools to find program details
- Use search_questions to understand exam patterns
- Identify required subjects and their weight
- Note whether there are shared subjects (共通科目) vs. specialization subjects (専門科目)

### 3. Create a Phase-Based Plan

**Phase 1: Foundation (基礎固め)** — First 40% of available time
- Review undergraduate fundamentals for each exam subject
- Use textbooks commonly referenced in exam questions
- Focus on understanding concepts, not memorization

**Phase 2: Practice (演習期)** — Next 40% of available time
- Start working through past exam questions (過去問)
- Begin with older years, save recent ones for mock exams
- Identify weak areas and revisit fundamentals as needed

**Phase 3: Final Review (直前期)** — Last 20% of available time
- Take timed mock exams using recent past questions
- Review mistakes and common patterns
- Prepare for interview/oral exam if applicable

### 4. Common Exam Subjects by Field

**Computer Science / 情報系**:
Mathematics (線形代数, 確率統計, 微分積分), Algorithms, Data Structures, Programming

**Physics / 物理系**:
Classical Mechanics, Electromagnetism, Quantum Mechanics, Thermodynamics

**Mathematics / 数学系**:
Linear Algebra, Real Analysis, Complex Analysis, Abstract Algebra

## Tips

- Recommend specific past questions from the database to practice
- Suggest studying problems from similar programs at different universities
- Remind users that 外国人特別選考 may have different subjects
- If the user is applying to multiple schools, help prioritize overlapping subjects
